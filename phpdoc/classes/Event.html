      Fetches logged events of a specific category.

      fetchEventsByCategory(string $category, string $age, int $status) : mixed[]

      Event categories include profile,groups,etc. The method accepts the category name and a timestamp value in 'YYYY-MM-DD HH:MM:SS' format. A status can be optionally specified as an argument to filter by the status of the events. The method fetches all events of the specified category logged since the specified timestamp.

      $eventset=$event->fetchEventsByCategory('profile','2012-12-01 00:00:00');

      Fetches logged events related to a particular entity.

      fetchEventsByEntity(int $orig, int $targ, string $age) : mixed[]

      The entity referred to here may be a group or user or any other defined entity. The method accepts the IDs of the corresponding source and/or targets of the events, corresponding to the entities. A timestamp is also accepted as a parameter to fetch event logs since a particular time. Timestamp follows the 'YYYY-MM-DD HH:MM:SS' format.

      $eventset=$event->fetchEventsByEntity(1,2,'2012-12-01 00:00:00');

      Fires an event explicitly.

      fireEvent(string $name, int $src, int $dest) : null

      The method accepts the name of the event as the first argument. The second is the source identifier and should be a valid user account ID. The third is the destination identifier and is optional. The method fires an event specified by the event name and logs it. The event name must be a registered event name. This engine comes with a set of registered events, which can be viewed by using the 'fetchEventNames' and 'fetchEventCategories' methods. Additional events may be registered by using the 'registerEvent' method.

      see \global\EventRegister::fetchEventNames()
      see \global\EventRegister::fetchEventCategories()
      see \global\EventRegister::registerEvent()
